# Plan: AI SDK Migration + useChat + Agent-Ready Architecture

> Generated by /plan on 2026-04-10 | Approach: Pragmatic Migration (with debate corrections)

## Feature Summary

Migrate all LLM calls from direct `@google/generative-ai` to the Vercel AI SDK (`ai` package), using `@ai-sdk/google` as the provider with the existing `GEMINI_API_KEY`. The web chat frontend migrates from manual fetch + useState to `useChat` from `@ai-sdk/react` with real-time token streaming. The Telegram bot path stays non-streaming. Structured output for `parseAddMessage` uses `generateText` with `Output.object()` and a Zod schema. The architecture separates streaming (web) and non-streaming (bot) paths so the web path can evolve into an agent later without changing the route handler or frontend.

  **Model helper** (replaces `getGenAI().getGenerativeModel()`):

  **`parseAddMessage(text: string): Promise<ParsedAdd>`:**
  - Use `generateText` with `Output.object({ schema: ParsedAddSchema })`
  - System prompt stays as `PARSE_SYSTEM` (unchanged content)
  - Wrap in try/catch — on any error, return the same fallback object currently on lines 47-54
  - The `output` property on the result contains the validated Zod object

  ```ts
  export async function parseAddMessage(text: string): Promise<ParsedAdd> {
    try {
      const { output } = await generateText({
        model,
        system: PARSE_SYSTEM,
        prompt: text,
        output: Output.object({ schema: ParsedAddSchema }),
      });
      if (!output) throw new Error('No output generated');
      return output;
    } catch {
      // Fallback: use raw input as title (preserves existing behavior)
      return {
        title: text.slice(0, 50),
        description: text.slice(0, 200),
        cost_tier: '$',
        cost_exact: null,
        category: 'other',
        tags: [],
      };
    }
  }
  ```

  **`answerQuery(ideas: DateIdea[], question: string): Promise<string>`:**
  - Use `generateText` with `system` + `prompt`
  - Return `result.text`
  - `ideasText` formatting stays identical
  - Signature unchanged — bot handlers need no modification

  ```ts
  export async function answerQuery(ideas: DateIdea[], question: string): Promise<string> {
    const ideasText = ideas
      .map(i => `- ${i.title} (${i.category}, ${i.cost_tier}${i.cost_exact ? `, ~$${i.cost_exact}` : ''}): ${i.description} [tags: ${i.tags.join(', ')}]`)
      .join('\n');

    const prompt = `Estas son las ideas guardadas:\n${ideasText}\n\nPregunta: ${question}`;

    const { text } = await generateText({
      model,
      system: QUERY_SYSTEM,
      prompt,
    });
    return text;
  }
  ```

  **`streamQueryResponse(ideas: DateIdea[], messages: UIMessage[])` (NEW):**
  - Accepts full `UIMessage[]` from the route handler
  - Builds `ideasText` the same way as `answerQuery`
  - Augments `QUERY_SYSTEM` with the ideas context
  - Calls `streamText` with `system` + `messages` (converted via `convertToModelMessages`)
  - Returns the `StreamTextResult` directly — do NOT await

  ```ts
  export function streamQueryResponse(ideas: DateIdea[], messages: UIMessage[]) {
    const ideasText = ideas
      .map(i => `- ${i.title} (${i.category}, ${i.cost_tier}${i.cost_exact ? `, ~$${i.cost_exact}` : ''}): ${i.description} [tags: ${i.tags.join(', ')}]`)
      .join('\n');

    const system = `${QUERY_SYSTEM}\n\nEstas son las ideas guardadas:\n${ideasText}`;

    return streamText({
      model,
      system,
      messages: convertToModelMessages(messages),
    });
  }
  ```

  Note: `convertToModelMessages` is async in AI SDK 6 — use `await` in the route handler or inside this function.

  **System prompts** (`PARSE_SYSTEM`, `QUERY_SYSTEM`): Keep content identical. These are module-level constants — no changes.

### Phase 3: Chat Route — Streaming Response
- [ ] Rewrite `app/api/chat/route.ts`

  **Before:**
  ```ts
  const { question } = await request.json();
  const answer = await answerQuery(ideas, question);
  return NextResponse.json({ answer });
  ```

  **After:**
  ```ts
  import { UIMessage } from 'ai';
  import { getAllIdeas } from '@/src/services/sheets';
  import { streamQueryResponse } from '@/src/services/ai';

  export const maxDuration = 55;

  export async function POST(request: Request) {
    const { messages }: { messages: UIMessage[] } = await request.json();

    if (!messages || !Array.isArray(messages) || messages.length === 0) {
      return new Response(JSON.stringify({ error: 'Se requiere el campo "messages"' }), {
        status: 400,
        headers: { 'Content-Type': 'application/json' },
      });
    }

    const ideas = await getAllIdeas();
    const result = streamQueryResponse(ideas, messages);

    return result.toUIMessageStreamResponse();
  }
  ```

  Key changes:
  - Request body: `{ question: string }` → `{ messages: UIMessage[] }`
  - Response: `NextResponse.json({ answer })` → `result.toUIMessageStreamResponse()`
  - Can remove `NextResponse` import (use plain `Response` for error)

### Phase 4: Frontend — useChat
- [ ] Rewrite `app/_components/ChatBox.tsx`

  Replace all manual state management (`question`, `answer`, `loading`, `error`) with `useChat` hook.

  ```tsx
  'use client';

  import { useChat } from '@ai-sdk/react';
  import { DefaultChatTransport } from 'ai';
  import { useState } from 'react';

  export default function ChatBox() {
    const { messages, sendMessage, status, stop, error } = useChat({
      transport: new DefaultChatTransport({ api: '/api/chat' }),
    });
    const [input, setInput] = useState('');

    const isLoading = status === 'submitted' || status === 'streaming';

    return (
      <div className="space-y-4">
        <form
          onSubmit={(e) => {
            e.preventDefault();
            if (input.trim()) {
              sendMessage({ text: input });
              setInput('');
            }
          }}
          className="flex gap-2"
        >
          <textarea
            value={input}
            onChange={(e) => setInput(e.target.value)}
            placeholder="Ej: ¿Qué opciones tenemos para un plan romántico y barato?"
            rows={2}
            required
            className="flex-1 rounded-lg border border-gray-300 px-3 py-2 text-sm focus:border-blue-500 focus:outline-none focus:ring-1 focus:ring-blue-500"
          />
          {isLoading ? (
            <button
              type="button"
              onClick={stop}
              className="self-end rounded-lg bg-red-600 px-4 py-2 text-sm font-medium text-white hover:bg-red-700"
            >
              Detener
            </button>
          ) : (
            <button
              type="submit"
              className="self-end rounded-lg bg-blue-600 px-4 py-2 text-sm font-medium text-white hover:bg-blue-700"
            >
              Preguntar
            </button>
          )}
        </form>

        {error && (
          <p className="text-sm text-red-600" role="alert">
            Error: {error.message}
          </p>
        )}

        <div className="space-y-2" aria-live="polite">
          {messages.map((message) => (
            <div
              key={message.id}
              className={`rounded-xl p-4 shadow-sm ${
                message.role === 'user' ? 'bg-blue-50' : 'bg-white'
              }`}
            >
              <p className="whitespace-pre-wrap text-sm">
                {message.parts
                  .filter((part) => part.type === 'text')
                  .map((part) => part.text)
                  .join('')}
              </p>
            </div>
          ))}
        </div>

        {isLoading && messages[messages.length - 1]?.role !== 'assistant' && (
          <div className="rounded-xl bg-white p-4 shadow-sm">
            <p className="text-sm text-gray-400">Buscando...</p>
          </div>
        )}
      </div>
    );
  }
  ```

  Key changes:
  - `useChat` manages messages array, status, error, and streaming
  - `sendMessage({ text: input })` replaces manual fetch
  - Messages rendered via `message.parts` (not a single string)
  - Stop button calls `stop()` during streaming
  - `status` replaces manual `loading` boolean
  - Full conversation history visible (multi-turn)

## Edge Cases

- [ ] Empty `messages` array to `/api/chat` — return 400 with error message
- [ ] `Output.object` validation failure — catch block returns fallback `ParsedAdd` with raw input as title
- [ ] Zero ideas in sheet — AI responds that there are no ideas; no crash
- [ ] Very long ideas list in system prompt — acceptable for two-person app; unlikely to hit token limits

### Config
- [ ] `GEMINI_API_KEY` missing — `@ai-sdk/google` throws at first AI call; documented in `.env.example`

### Performance
- [ ] `getAllIdeas()` runs on every chat POST — acceptable for two-person app
- [ ] `convertToModelMessages` is async in AI SDK 6 — must `await` it

### Accessibility
- [ ] `status === 'submitted' || 'streaming'` — submit button replaced with stop button, shows loading state
- [ ] All UI text remains in Spanish
- [ ] `aria-live="polite"` on message container for screen reader updates

- `maxDuration = 55` must be retained on the chat route

- Do not import anything from `src/bot/` in web/Next.js code

## Risks & Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| `Output.object` returns null on malformed Gemini output | Bot `/add` silently fails | try/catch with fallback object (same fallback as current code) |
| `useChat` API differs from expected (v6 breaking changes) | Frontend doesn't compile | All APIs verified from `node_modules/ai/docs/` — `sendMessage`, `status`, `parts` confirmed |
| `@ai-sdk/google` default env var is `GOOGLE_GENERATIVE_AI_API_KEY`, not `GEMINI_API_KEY` | AI calls fail at runtime | Pass `apiKey` explicitly via `google()` constructor or `createGoogleGenerativeAI({ apiKey })` |
| `convertToModelMessages` is async in v6 (was sync in v5) | Type error or runtime crash | `await` the call — verified in migration guide |
| Route + frontend must update atomically (different request/response shape) | Broken chat if deployed partially | Both changes in same commit/PR |
