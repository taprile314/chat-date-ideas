## Implementation Summary

Migrated the web chatbot and Telegram bot to Vercel Durable Workflows using `DurableAgent` from `@workflow/ai/agent`. The agent uses Gemini 2.5 Flash with 4 tools (searchIdeas, addIdea, listAllIdeas, getRandomIdea) backed by Google Sheets. Web chat streams via `WorkflowChatTransport` with automatic reconnection; Telegram uses a sync bridge that awaits the full workflow result via `run.returnValue`.

## Code Quality Notes (advisory)

- `workflows/chat/index.ts:25` — The `async () => model` factory is required because DurableAgent's type expects `string | (() => Promise<CompatibleLanguageModel>)` and there's a V2/V3 type mismatch with `@ai-sdk/google`. A future `@workflow/ai` release may fix this.
- `workflows/chat/tools.ts:74` — `listAllIdeas()` silently truncates at 100 rows. At current scale this is fine; consider adding a truncation note if the sheet grows.
- `app/api/agent/route.ts` — The response extraction walks model messages manually. If WDK exposes a typed text accessor in a future release, simplify this.
