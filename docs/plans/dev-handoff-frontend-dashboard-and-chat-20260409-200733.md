## Implementation Summary

Migrated the existing Telegram bot project to a Next.js 16 App Router app with Tailwind CSS v4. Added a password-protected web dashboard with full CRUD for date ideas (add via AI parsing, edit structured fields, delete with confirmation) and a chat interface that queries saved ideas via the Gemini AI agent. The Telegram webhook (`api/webhook.ts`) is preserved as a standalone Vercel serverless function alongside the Next.js app.

## Testing Strategy

- **Phase 1**: `bun run build` succeeds; `bun run dev` shows Next.js page; Telegram webhook responds (curl test)
- **Phase 2**: Access `/dashboard` without cookie → redirect to `/login`; login with correct password → access; login with wrong password → error; logout → redirect to login
- **Phase 3**: curl GET `/api/ideas` → JSON list of ideas; curl POST `/api/ideas` with text → idea created in Sheet; curl PATCH `/api/ideas/[id]` → idea updated; curl DELETE `/api/ideas/[id]` → idea deleted; curl POST `/api/chat` → agent response
- **Phase 4**: Dashboard shows ideas; filters work; full add flow; full edit flow; delete with confirmation
- **Phase 5**: Chat sends question, shows loading, displays Spanish response
- **Phase 6**: Mobile responsive; webhook still works; preview deploy OK
- **Regression**: Send `/add` and `/query` commands to the Telegram bot after each deploy — verify nothing broke
