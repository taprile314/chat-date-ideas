# Plan: Telegram Date Ideas Bot

> Generated by /plan on 2026-04-09 | Approach: Pragmatic Serverless Bot

## Feature Summary

A Telegram bot that lets a couple save and query date ideas via chat commands with natural language parsing. The bot lives in a shared Telegram group where both partners can `/add` ideas in Spanish (e.g., "quiero ir al restaurante La Mar, como $50 por persona") and Claude Haiku automatically extracts the title, cost, category, and tags. Ideas are stored in a Google Sheet (visible to both partners). To find ideas, either partner uses `/query` (e.g., "algo romantico y barato este fin de semana") and Claude Sonnet reads the Sheet, filters/ranks, and responds conversationally in Spanish.

The bot is deployed as a Vercel serverless function using Telegram's webhook mode — no always-on server needed.

## Feature Type

integration + shared-state, mobile-first, serverless, nlp-driven, spanish-language

## Implementation Order

### Phase 1: Foundation (Types + Config)

- [ ] Initialize project with `npm init` and install dependencies (`@anthropic-ai/sdk`, `googleapis`, `typescript`) -> `package.json`
- [ ] Create TypeScript config -> `tsconfig.json`
- [ ] Define all interfaces: `DateIdea`, `ParsedAdd`, `SheetRow`, `CostTier`, `TelegramUpdate`, `TelegramMessage` -> `src/types/idea.ts`
- [ ] Implement env var loader with validation (throw on missing vars) -> `src/config.ts`
- [ ] Create `.env.example` with all required vars documented
- [ ] Create `.gitignore`

### Phase 2: Services (Sheets + Claude)

- [ ] Implement Google Sheets client using service account auth -> `src/services/sheets.ts`
  - `appendIdea(idea: DateIdea): Promise<void>` — appends one row
  - `getAllIdeas(limit?: number): Promise<DateIdea[]>` — reads all rows (default limit 100)
- [ ] Create and run setup script to initialize Sheet with headers -> `scripts/setup-sheet.ts`
- [ ] Test Sheets connection: run `setup-sheet.ts`, verify headers appear in Google Sheet
- [ ] Implement Claude service -> `src/services/claude.ts`
  - `parseAddMessage(text: string): Promise<ParsedAdd>` — Haiku, returns structured JSON
  - `answerQuery(ideas: DateIdea[], question: string): Promise<string>` — Sonnet, returns Spanish response
- [ ] Test Claude parsing: run `parseAddMessage` with sample Spanish inputs, verify JSON output

### Phase 3: Bot Handlers

- [ ] Implement Telegram reply wrapper with markdown formatting -> `src/bot/reply.ts`
- [ ] Implement `/help` handler (static Spanish message listing all commands) -> `src/bot/handlers/help.ts`
- [ ] Implement `/add` handler -> `src/bot/handlers/add.ts`
  - Extract text after `/add` command
  - Guard: if empty, reply with usage hint
  - Call `parseAddMessage()` to get structured data
  - Call `appendIdea()` to save to Sheet
  - Reply with confirmation: "Guardado: [title] ([category], [cost_tier])"
- [ ] Implement `/query` handler -> `src/bot/handlers/query.ts`
  - For `/query <text>`: pass text as the question
  - For `/list`: use implicit prompt "muéstrame todas las ideas"
  - For `/random`: use implicit prompt "dame una idea aleatoria"
  - Call `getAllIdeas()` to read Sheet
  - Guard: if 0 ideas, reply with empty-state message
  - Call `answerQuery()` with ideas + question
  - Reply with Claude's Spanish response
- [ ] Implement command router -> `src/bot/router.ts`
  - Priority: `/add` > `/list` > `/random` > `/query` or `/buscar` > `/help` or `/ayuda`
  - Group chat: ignore free text unless bot is @mentioned
  - DM: treat free text as `/query`

### Phase 4: Webhook + Deploy

- [ ] Implement Vercel serverless function -> `api/webhook.ts`
  - Validate `X-Telegram-Bot-Api-Secret-Token` header
  - Parse JSON body as `TelegramUpdate`
  - Call router
  - Always return HTTP 200 (even on error)
- [ ] Create Vercel config -> `vercel.json`
- [ ] Deploy to Vercel: `vercel --prod`
- [ ] Set Telegram webhook: `curl -X POST https://api.telegram.org/bot<TOKEN>/setWebhook -d url=https://<PROJECT>.vercel.app/api/webhook -d secret_token=<SECRET>`
- [ ] Test full flow: send `/help`, `/add`, `/query` in Telegram group
- [ ] Add partner to group and verify shared access

## Google Sheets Schema

**Sheet name:** `ideas`

| Column | Key | Type | Notes |
|--------|-----|------|-------|
| A | `id` | string | UUID v4 — for future deletion/edit |
| B | `created_at` | ISO datetime | Server time at insertion |
| C | `added_by` | string | Telegram `first_name` of sender |
| D | `title` | string | Claude-extracted short title (max 50 chars) |
| E | `description` | string | Full detail, cleaned from original text (max 200 chars) |
| F | `cost_tier` | string | `free`, `$`, `$$`, `$$$` |
| G | `cost_exact` | number or empty | Extracted number if user provided one |
| H | `category` | string | Primary: `restaurant`, `outdoors`, `movie`, `activity`, `travel`, `other` |
| I | `tags` | string | Comma-separated secondary tags |
| J | `raw_input` | string | Original user message — never modified, for debugging |

Row 1 is the frozen header row (written by `setup-sheet.ts`). Data starts at row 2.

## Claude API Prompts

### Haiku — Parse `/add` message

**System prompt:**
```
Eres un asistente que extrae datos estructurados de ideas de citas/salidas.
Devuelve SOLO un JSON válido sin texto adicional, con este esquema:
{
  "title": "string (máx 50 caracteres, español)",
  "description": "string (máx 200 caracteres, español)",
  "cost_tier": "free | $ | $$ | $$$",
  "cost_exact": number | null,
  "category": "restaurant | outdoors | movie | activity | travel | other",
  "tags": ["string"]
}
Si no se menciona costo, usa cost_tier "$" y cost_exact null.
Si no puedes determinar la categoría, usa "other".
```

### Sonnet — Answer `/query`

**System prompt:**
```
Eres un asistente de pareja que ayuda a encontrar ideas para salir.
Responde en español, de forma corta y amigable.
Cuando hay muchas opciones, muestra máximo 3-4.
Si preguntan por presupuesto, filtra por cost_tier y cost_exact.
Si no hay ideas que coincidan, sugiere que agreguen más con /add.
Formato: usa emojis para categorías, nombre en negrita, costo entre paréntesis.
```

## Functional Requirements

| Requirement | Trigger | Expected Behavior | Timing | Reversible? | Accessibility |
|-------------|---------|-------------------|--------|-------------|---------------|
| Save idea | `/add <text>` | Claude Haiku parses text into structured JSON, row appended to Sheet, bot confirms with title + category in Spanish | < 5s target | No delete in MVP; raw data in Sheet for manual edit | Plain text response |
| Query ideas | `/query <text>` or free DM | All Sheet rows sent to Sonnet, returns filtered + formatted list in Spanish | < 10s target | N/A | Short response, 3-4 items max |
| List all | `/list` | Same as query with "all ideas" prompt — grouped by category | < 10s target | N/A | Truncated if > 10 ideas |
| Random idea | `/random` | Claude picks one idea at random, full details | < 10s target | N/A | Single card format |
| Help | `/help` or `/ayuda` | Static message listing all commands in Spanish | Instant | N/A | Text-only |
| Group noise suppression | Free text in group | Bot ignores unless @mentioned | Instant | N/A | No response = no confusion |

## Edge Cases

### Shared Access
- [ ] Both users add ideas simultaneously — Sheets `append` is atomic per call, no collision risk
- [ ] Responses always in Spanish regardless of user Telegram locale (system prompt enforces)
- [ ] Bot removed from group — webhook delivery stops after ~1h; re-adding resolves

### Data & Content
- [ ] Empty `/add` with no payload — handler replies with usage hint: "Usa /add seguido de tu idea. Ejemplo: /add ir al cine este viernes"
- [ ] Very long description — Claude trims to 200 chars description / 50 chars title
- [ ] Cost in non-standard format — Claude maps to `cost_tier`, stores number without currency symbol in `cost_exact`
- [ ] No ideas saved yet on first `/query` — friendly message: "Aun no tienen ideas guardadas. Usa /add para agregar la primera."
- [ ] Duplicate ideas — no deduplication in MVP; acceptable for a couple's usage

### Serverless
- [ ] Cold start on first request — adds ~1-2s; acceptable within 55s budget
- [ ] Vercel Hobby 10s timeout — `/query` may timeout if Sonnet + Sheets takes too long; mitigation: upgrade to Pro ($20/mo) or pre-filter rows before sending to Claude
- [ ] Missing webhook secret header — function returns 401 and logs warning

### Google Sheets API
- [ ] Service account auth (not OAuth) — no refresh token issues
- [ ] Quota (100 req/100s) — each command uses 1-2 API calls; 2 users are far below limit
- [ ] Sheet accidentally deleted — no backup in MVP; export CSV periodically as manual mitigation

### Performance
- [ ] 500+ rows in Sheet — cap `getAllIdeas` to last 100 rows for queries to keep prompt size manageable
- [ ] Sheets API auth client instantiated at module level (not per-request) to avoid re-auth on warm Vercel invocations

## Constraints

- Vercel Hobby tier: 10s function timeout (Pro: 60s) — `/query` latency is the primary risk
- Claude API costs: Haiku is cheap (~$0.001/call), Sonnet is pricier (~$0.01/call) — acceptable for personal use
- Google Sheets API: 100 requests per 100 seconds per user — not a concern for 2 users
- No database — Google Sheets is the only persistence layer; no ACID transactions
- No user authentication beyond Telegram group membership
- All bot responses in Spanish

## Risks & Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| Vercel Hobby 10s timeout on `/query` | High — queries fail silently | Upgrade to Pro ($20/mo), or send "Buscando..." reply first, then edit message with result |
| Claude returns invalid JSON on `/add` | Medium — idea not saved properly | Fallback: save with `title` = first 50 chars of raw input, `category` = "other" |
| Google Sheets API outage | Medium — all reads/writes fail | Return friendly error message; ideas are logged to Vercel function logs for recovery |
| Claude API outage | Medium — parsing and queries fail | Return "Intenta de nuevo en un momento" message |
| Sheet grows very large (1000+ rows) | Low — slow queries, large prompts | Cap `getAllIdeas` to 100 rows; future: archive old ideas to separate sheet |
| Unauthorized users in group | Low — anyone in group can use bot | Acceptable for a couple; add allowlist if group grows |
| Telegram webhook delivery failures | Low — ideas lost | Telegram retries on non-200; function always returns 200, so monitor Vercel logs |
